# MG Enterprise — Admin Users Configuration
# ============================================================
# LOCATION: This file lives in the app data directory, NOT here in the project.
#   macOS: ~/Library/Application Support/com.mgenterprise.clarix/admin_users.txt
#
# Each non-blank, non-comment line defines one user.
# Format (comma-separated, 5 fields):
#
#   email, password, name, role, is_developer
#
# Fields:
#   email          - login email address (must be unique, case-insensitive)
#   password       - plain-text password (hashed automatically on save)
#   name           - display name shown in the UI
#   role           - one of: SuperAdmin, SiteSuperAdmin, Manager, User
#   is_developer   - true or false  (developers see Add Screen button etc.)
#
# How password changes work:
#   1. Edit the password field in this file
#   2. Restart the app
#   3. The app detects the change via admin_users_applied.txt (internal)
#      and re-hashes + saves the new password automatically
#
# Lines starting with # are comments and are ignored.
# Blank lines are ignored.
# ============================================================
